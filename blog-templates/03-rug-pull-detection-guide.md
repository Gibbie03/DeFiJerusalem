### What Is a Rug Pull? (250 words)

**Definition**: When developers abandon a project and run away with investors' funds

**Types of Rug Pulls**:

1. **Liquidity Pulling**
   - Most common type
   - Developers remove all liquidity from DEX
   - Token becomes worthless instantly

2. **Limiting Sell Orders (Honeypot)**
   - Can buy but not sell
   - Hidden in smart contract

3. **Dumping Dev Tokens**
   - Team holds large allocation
   - Dumps on retail investors
   - Price crashes

### 15 Red Flags of a Rug Pull (800 words)

**Create detailed numbered list**:

### 🚩 Red Flag #1: Anonymous Team
- No doxxed team members
- Stock photos or AI-generated avatars
- No LinkedIn profiles
- No previous project history

**How to Check**:
- Reverse image search team photos
- Look up on LinkedIn
- Check GitHub contributions
- Verify on JERUSALEM (team verification score)

---

### 🚩 Red Flag #2: No Locked Liquidity
- Liquidity can be removed anytime
- Or very short lock period (<6 months)

**How to Check**:
- Check liquidity lock on Unicrypt/Team Finance
- Minimum 1 year lock for legitimate projects
- JERUSALEM shows liquidity lock status

---

### 🚩 Red Flag #3: Centralized Ownership
- Single wallet controls contract
- No multisig wallet
- No governance structure

**How to Check**:
- View contract on Etherscan
- Look for "owner" address
- Check if multisig (3+ signers)
- JERUSALEM detects centralized control

---

### 🚩 Red Flag #4: No Timelock on Admin Functions
- Developers can make changes instantly
- No 24-48 hour delay for security

**How to Check**:
- Review contract code
- Look for timelock modifier
- JERUSALEM scans for timelock protection

---

### 🚩 Red Flag #5: Unrealistic Promises
- "10,000% APY guaranteed!"
- "Can't lose money"
- "Next 100x project"

**Reality Check**:
- Nothing is guaranteed in crypto
- High APY = high risk
- Sustainable protocols have reasonable returns
