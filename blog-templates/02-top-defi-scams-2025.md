## Scam #1: Wallet Drainer Attacks (300 words)

**What It Is**:
- Malicious smart contracts that empty your entire wallet
- Uses fake airdrops, phishing sites, malicious NFTs

**How It Works**:
1. Victim visits fake protocol site (e.g., uniswaap.com)
2. Connects wallet
3. Signs "approval" transaction
4. Drainer empties all tokens instantly

**Real Example**: 
- Aster-Dex scam (aster-dex.lol)
- $X million stolen from X,XXX victims
- Used .lol domain (90%+ scam indicator)

**Red Flags**:
- Suspicious domain (.lol, .tk, .ml)
- Urgent "claim now" messaging
- Requests unlimited token approval

**How JERUSALEM Detects**:
- Malicious TLD scanning
- Phishing language detection
- Contract verification
- Automatic blacklisting

**Protection Tips**:
- Always check domain carefully
- Scan on JERUSALEM before connecting

- Never approve unlimited amounts

## Scam #2: Rug Pull Schemes (300 words)

**What It Is**:
- Developers drain liquidity pool and disappear
- Token becomes worthless overnight

**How It Works**:

2. Heavy marketing, fake partnerships
3. Lock liquidity (sometimes fake lock)

5. Developers vanish

**Red Flags**:
- Anonymous team

- Centralized control (no multisig)
- No timelock on smart contracts
- Unrealistic APY promises

**How JERUSALEM Detects**:
- Liquidity lock verification
- Multisig checking
- Timelock analysis
- Whale concentration monitoring
- Ownership analysis

**Protection Tips**:
- Check team credentials
- Verify liquidity is locked (minimum 1 year)
- Look for multisig wallet (3+ signers)
- Scan on JERUSALEM for red flags
- Never invest more than you can lose

## Scam #3: Fake Token Airdrops (250 words)

**What It Is**:
- Scammers promise "free tokens" from popular protocols
- Actually steals your funds when you "claim"

**How It Works**:

3. Connect wallet to fake site
4. Sign "claim" transaction
5. Wallet drained

**Real Example**:
- Fake Uniswap airdrop (claim-uniswap.org)
- Impersonated official announcement
- Targeted users with genuine UNI holdings

**Red Flags**:
- Unsolicited airdrop notifications
- Urgent "limited time" deadlines
- Suspicious domain names
- Requests for seed phrase/private key

**How JERUSALEM Detects**:
- Imposter protocol detection
- Phishing domain analysis
- Social engineering pattern matching

**Protection Tips**:
- Never trust unsolicited messages
- Verify airdrops on official Twitter/Discord
- Check domain carefully
- Legitimate airdrops NEVER ask for seed phrases

## Scam #4: Honeypot Tokens (250 words)

4. Scammer dumps at peak

**Real Example**:

- Price went from $0.01 to $2,861
- Investors couldn't sell
- Developers cashed out, disappeared

**Red Flags**:
- Can't find sell transactions on blockchain explorer
- Suspicious code in contract
- Anonymous developers
- Low liquidity
- Social media hype but no substance

**How JERUSALEM Detects**:
- Smart contract code analysis
- Hidden function detection
- Sell restriction scanning
- Honeypot pattern matching

**Protection Tips**:
- Check contract on Etherscan/BSCScan

- Use honeypot detection tools
- Scan on JERUSALEM before buying

## Scam #5: Phishing Attacks (250 words)

**What It Is**:
- Fake websites/emails that steal wallet credentials
- Impersonates legitimate DeFi protocols

**How It Works**:
1. Create pixel-perfect copy of popular protocol
2. Register similar domain (typosquatting)
3. Lure victims via ads, social media
4. Victim enters seed phrase/private key
5. Wallet compromised

**Real Examples**:
- Fake MetaMask sites
- Phishing emails claiming "security update required"
- Twitter DMs from "support team"

**Red Flags**:
- Slight domain misspellings (metaomask.io)
- HTTPS padlock doesn't mean safe!
- Requests for seed phrase
- Urgent security warnings
- Poor grammar/spelling

**How JERUSALEM Detects**:
- Imposter domain detection (90+ verified protocols)
- Typosquatting pattern analysis
- Visual clone detection
- Phishing language scanning

**Protection Tips**:
- Bookmark official sites
- Double-check URLs before connecting
- NEVER share seed phrase
- Enable 2FA where possible
- Use hardware wallet

## Scam #6: Ponzi/Pyramid Schemes (250 words)

**What It Is**:
- "Investment" that pays old investors with new investors' money
- Unsustainable, eventually collapses

**How It Works**:
1. Promise guaranteed high returns (e.g., 5% daily)
2. Pay early investors with new deposits
3. Recruit more investors (pyramid structure)
4. System collapses when new money stops
5. Founders disappear with funds

**Real Examples**:
- BitConnect (infamous 2017 collapse)
- PlusToken ($2.9 billion stolen)
- Various "cloud mining" scams

**Red Flags**:
- Guaranteed returns (nothing is guaranteed!)
- Recruitment bonuses
- No clear revenue model
- Anonymous team
- Pressure to "act fast"

**How JERUSALEM Detects**:
- Ponzi language detection ("guaranteed", "risk-free")
- Unsustainable return modeling
- Pyramid structure analysis
- Historical scam pattern matching

**Protection Tips**:
- If it sounds too good to be true, it is
- Research team thoroughly
- Ask: "Where do profits come from?"
- No legitimate investment guarantees returns
- Scan on JERUSALEM before investing

## Scam #7: Oracle Manipulation Attacks (250 words)

**What It Is**:
- Exploiting price oracles to manipulate DeFi protocol prices
- Technical but devastating

**How It Works**:
1. Find protocol with vulnerable oracle
2. Flash loan large amount
3. Manipulate price on target DEX
4. Execute trades at manipulated price
5. Profit and return flash loan

**Real Examples**:
- Cream Finance hack ($130M+)
- Harvest Finance attack ($24M)
- Various flash loan attacks

**Red Flags**:
- Single oracle source (no redundancy)
- No timelock on price updates
- Low liquidity pools as price source
- Lack of manipulation resistance

**How JERUSALEM Detects**:
- Oracle vulnerability scanning
- Price manipulation risk assessment
- Flash loan attack history
- Security audit verification

**Protection Tips**:
- Check if protocol uses Chainlink (safer)
- Verify oracle redundancy
- Look for recent security audits
- Avoid protocols with history of oracle issues
